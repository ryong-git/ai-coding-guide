## 💡 KB가 없을 때 vs 있을 때

### ❌ KB 없이 AWS 공식 문서만 사용

**한계:**
1. **베스핀 특화 정보 없음**
   - "왜 고객이 직접 못하나요?" → 답변 불가
   - Security Group 변경 절차 → 일반론만

2. **내부 프로세스 모름**
   - SLA 안내 불가
   - 승인 절차 설명 불가
   - 담당자 에스컬레이션 기준 모름

3. **고객사 특성 반영 안 됨**
   - 버거킹 JEUS 재시작 → 일반적 재시작 방법만
   - LG SQL Storage → 일반적 스토리지 확장만

**결과:**
- 🔴 답변이 추상적이고 불완전
- 🔴 고객이 "베스핀에 직접 요청해야 하나요?" 재질문
- 🔴 결국 담당자 개입 필요 → **자동화 실패**

### ✅ KB 적용 시

**베스핀 전용 정보 포함:**
1. **"왜?"에 대한 답변**
   ```
   고객님께서는 AWS Organizations 멤버 계정으로,
   네트워크 보안은 베스핀글로벌에서 중앙 관리합니다.
   변경 관리 절차에 따라 저희가 처리해드리겠습니다.
   ```

2. **구체적인 프로세스**
   ```
   Security Group 변경 처리 시간:
   - 일반: 1영업일 이내
   - 긴급: 4시간 이내
   
   필요 정보:
   - 출발지 IP
   - 목적지 리소스
   - 포트 번호
   ```

3. **고객사별 맞춤 정보**
   ```
   버거킹 RIMS 서버 JEUS 재시작:
   - 서버: prod-rims-01
   - 재시작 스크립트: /opt/jeus/restart.sh
   - 영업 시간외 권장
   - 예상 다운타임: 3분
   ```

**결과:**
- 🟢 완전하고 실행 가능한 답변
